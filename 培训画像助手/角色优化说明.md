### 4. 自动更新机制 ✅

已创建知识库自动更新脚本：`知识库自动更新.py`

**功能特性**：
- 自动检测knowledgebase目录下的文件变化（新增、修改、删除）
- 使用文件哈希值精确识别文件变更
- 自动更新知识库索引文件
- 保存文件状态，支持增量更新

**使用方法**：
```bash
# 激活虚拟环境
source venv/bin/activate

# 运行自动更新脚本
python 知识库自动更新.py
```

**自动更新流程**：
1. 扫描knowledgebase目录下的所有文件
2. 计算文件哈希值，与上次状态对比
3. 检测新增、修改、删除的文件
4. 自动更新知识库索引文件
5. 保存新的文件状态

## 使用建议

### 1. 定期运行自动更新
建议在以下情况运行自动更新脚本：
- knowledgebase目录下有新文件添加时
- 政策文件更新后
- 定期维护时（如每月一次）

### 2. 查看知识库索引
当需要了解知识库中有哪些文件时，可以：
- 查看`knowledge/knowledgebase-index.knowledge.md`
- 或运行`知识库整合.py`重新生成索引

### 3. 使用知识库
角色激活后，会自动加载所有知识库内容。在需要查询时：
- 政策依据：查阅对应的政策文件
- 评价标准：参考评价指标体系文档
- 数据格式：查看数据示例文件
