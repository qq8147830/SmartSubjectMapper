### 2. 培训数据集成（张磊的200条培训记录）

#### 2.1 数据来源
- 源文件：`培训记录数据_已标注1.0.csv`（200条记录）
- 数据转换：通过`数据处理1.0.py`处理并生成`培训数据数组1.0.json`
- 数据格式：已转换为JavaScript数组格式，包含完整字段

#### 2.2 数据字段
每条培训记录包含：
- `id`: 培训记录ID（1-200）
- `title`: 培训项目名称
- `level`: 培训级别（国家级培训、省级培训、市级培训、校级培训、其它）
- `year`: 完成年份
- `month`: 完成月份
- `certificate`: 是否有证书（true/false）
- `hours`: 培训学时
- `status`: 培训状态（已结束/进行中）
- `category1`: 一级维度标签
- `category2`: 二级维度标签
- `dimension`: 培训维度

#### 2.3 数据分布统计
- **专业理念与师德**: 包含`ethics-cultivation`和`ethics-concept`标签的记录
- **专业知识**: 主要为`knowledge-subject`标签的记录
- **专业能力**: 包含`ability-design`、`ability-management`、`ability-research`、`ability-development`等标签的记录
- **数字素养**: 主要为`digital-tech`标签的记录

### 3. 前端数据存储和调用机制

#### 3.1 localStorage存储机制
实现了完整的前端数据存储机制：

**存储键名：**
- `training_profile_data_1.0`: 培训数据
- `training_tag_config_1.0`: 标签配置

**数据加载优先级：**
1. 优先从`localStorage`加载（如果存在）
2. 其次从外部JS文件加载（`training_data_js1.0.js`）
3. 再次从JSON文件加载（`培训数据数组1.0.json`）
4. 最后使用内嵌的默认数据（10条示例数据）

**数据保存：**
- 数据加载成功后自动保存到`localStorage`
- 标签配置保存到`localStorage`
- 支持数据持久化，刷新页面后数据仍然可用

#### 3.2 数据加载函数
- `loadTrainingData()`: 从localStorage加载培训数据
- `saveTrainingData(data)`: 保存培训数据到localStorage
- `loadTagConfig()`: 从localStorage加载标签配置
- `saveTagConfig(config)`: 保存标签配置到localStorage
- `loadFullTrainingData()`: 从外部文件加载完整数据（200条）
- `loadFromJSON()`: 从JSON文件加载数据

## 四、使用说明

### 1. 预览报告
1. 打开`培训画像报告1.0.html`文件
2. 页面加载后会自动从localStorage或外部文件加载完整数据
3. 查看控制台日志，确认数据加载状态

### 2. 测试标签筛选
1. 点击一级标签（如"专业能力"）
2. 查看二级标签是否正确显示
3. 点击二级标签（如"教学设计"）
4. 验证筛选结果是否正确显示对应的培训主题

### 3. 数据更新
- 如需更新数据，修改`培训数据数组1.0.json`或`training_data_js1.0.js`
- 清除浏览器localStorage后刷新页面，将重新加载最新数据
- 或直接调用`loadFullTrainingData()`函数重新加载
