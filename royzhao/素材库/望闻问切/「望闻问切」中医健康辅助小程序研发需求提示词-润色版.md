# 「望闻问切」中医健康辅助小程序研发需求提示词（润色版）

## 一、项目定位与战略目标

### 1.1 产品定位
**核心定位**：健康辅助工具（非医疗设备），提供中医体质分析、脉象分析及个性化健康建议，仅作健康参考，不可替代专业医疗诊断。

**战略目标**：
- **短期目标**：建立基于HRV技术的中医体质识别系统，提供基础健康参考服务
- **中期目标**：完善复合脉象识别能力，形成"望闻问切"四诊合参的数字化健康辅助体系
- **长期目标**：构建中医健康管理生态，成为用户日常健康监测与调理的智能助手

### 1.2 目标用户群体
- **主要用户**：关注中医养生的健康人群、亚健康人群
- **次要用户**：慢性病管理人群（需在医生指导下使用）、中医爱好者
- **使用场景**：日常健康监测、体质调理参考、健康生活方式指导

### 1.3 核心价值主张
- **科学性**：基于权威中医理论（王琦教授九分法）与现代HRV技术结合
- **便捷性**：无需专业设备，仅用手机即可完成检测
- **个性化**：因人制宜，提供定制化健康建议
- **安全性**：严格遵循法规要求，明确免责声明

## 二、核心理论依据

### 2.1 中医体质学基础理论
**主要依据**：
- **核心标准**：王琦教授"中医体质九分法"（来源：《9种基本中医体质类型的分类及其诊断表述依据》，北京中医药大学学报 2005,28(4):1-8）
- **行业标准**：《中医体质分类与判定标准》（中华中医药学会标准 ZYYXH/T157-2009）

**九种基本体质类型**：
1. 平和质（健康体质）
2. 气虚质
3. 阳虚质
4. 阴虚质
5. 痰湿质
6. 湿热质
7. 血瘀质
8. 气郁质
9. 特禀质

### 2.2 延伸理论支撑
- **基因组学研究**：9种中医体质的基因组学研究（中华中医药杂志 2014、2020年相关研究）
- **体质兼夹理论**：一般人群体质兼夹状况分析，处理复合体质情况
- **脉象学理论**：传统中医脉学经典（《脉经》《濒湖脉学》等）与现代HRV技术的融合

### 2.3 技术理论依据
- **HRV分析理论**：心率变异性（Heart Rate Variability）作为自主神经系统功能的客观指标
- **信号处理理论**：PPG（光电容积脉搏波）信号采集与处理技术
- **机器学习理论**：防过拟合优化算法在医疗AI中的应用（参考相关医疗AI研究文献）

## 三、核心功能需求

### 3.1 中医体质分析功能

#### 3.1.1 九种体质识别
**技术实现**：
- **数据来源**：基于心率变异性（HRV）数据分析
- **核心参数**：RMSSD、pNN50、SDNN、LF/HF比值等HRV时域与频域参数
- **识别算法**：结合HRV特征与中医体质理论，建立体质识别模型
- **输出结果**：识别用户所属的9种基本体质类型（支持单一体质或复合体质）

**功能要求**：
- 支持体质倾向度评分（0-100分），显示主要体质及次要体质
- 提供体质识别置信度评估
- 记录历史体质变化趋势

#### 3.1.2 个性化健康建议
**建议维度**（遵循中医"因人制宜"原则）：
1. **饮食调理**
   - 推荐适宜食物与禁忌食物
   - 提供食疗方推荐
   - 饮食时间与方式建议

2. **生活作息**
   - 作息时间建议
   - 睡眠质量改善方案
   - 季节养生指导

3. **运动锻炼**
   - 推荐运动类型与强度
   - 运动时间与频率建议
   - 传统养生功法推荐（如八段锦、太极拳等）

4. **情绪调节**
   - 情志调养建议
   - 心理疏导方法
   - 音乐、冥想等辅助方法

**个性化程度**：
- 根据体质类型提供差异化建议
- 考虑用户年龄、性别、季节等因素
- 支持用户反馈优化建议内容

### 3.2 脉象分析功能

#### 3.2.1 复合脉象识别
**技术实现**：
- **数据融合**：传统中医脉学理论 + 现代HRV技术
- **识别类型**：弦滑、细数、洪数、沉迟、浮数等常见复合脉象
- **分析方法**：
  - HRV频域分析（LF、HF、VLF频段特征）
  - 时域特征提取（脉率、脉律、脉力等）
  - 结合传统脉象28脉特征进行模式匹配

**功能要求**：
- 支持实时脉象监测（30秒-2分钟）
- 提供脉象特征可视化展示
- 记录历史脉象数据，支持趋势分析

#### 3.2.2 证候分析
**分析依据**：
- **理论基础**：中医辨证论治理论
- **分析流程**：脉象特征 → 证候推断 → 调理建议
- **输出内容**：
  - 证候类型判断（如：肝郁气滞、脾虚湿盛等）
  - 证候严重程度评估
  - 对应调理建议（中药、针灸、推拿等方向性建议，非具体处方）

**注意事项**：
- 明确标注"仅供参考，不可替代专业中医诊断"
- 建议用户结合其他症状综合判断
- 引导用户咨询专业中医师

### 3.3 数据记录与追踪功能
- **健康档案**：记录用户体质、脉象历史数据
- **趋势分析**：展示体质、脉象变化趋势图表
- **健康报告**：生成周期性健康分析报告（周报/月报）
- **数据导出**：支持健康数据导出，便于用户保存或提供给医生参考

## 四、技术实现要求

### 4.1 HRV检测技术

#### 4.1.1 信号采集
**硬件要求**：
- **设备**：手机摄像头 + 闪光灯（或LED补光灯）
- **采集部位**：指尖（推荐）或面部（需考虑环境光干扰）
- **采集时长**：建议60-120秒，确保数据质量

**技术实现**：
- **PPG信号采集**：利用光电容积脉搏波技术，通过摄像头捕捉血流变化
- **信号预处理**：
  - 去噪处理（环境光干扰、运动伪影等）
  - 信号滤波（带通滤波，保留0.04-0.5Hz频段）
  - 峰值检测与R-R间期提取

#### 4.1.2 HRV参数计算
**时域参数**：
- **SDNN**：全部正常R-R间期的标准差（反映整体HRV水平）
- **RMSSD**：相邻R-R间期差值的均方根（反映副交感神经活性）
- **pNN50**：相邻R-R间期差值>50ms的百分比（反映副交感神经活性）

**频域参数**：
- **LF（0.04-0.15Hz）**：低频功率，反映交感与副交感神经共同调节
- **HF（0.15-0.4Hz）**：高频功率，主要反映副交感神经活性
- **LF/HF比值**：反映自主神经平衡状态

**非线性参数**（可选）：
- **SD1/SD2**：Poincaré散点图参数
- **样本熵**：反映心率序列的复杂性

#### 4.1.3 数据质量评估
- **信号质量评分**：评估采集信号的信噪比
- **异常值检测**：识别并处理异常R-R间期
- **数据有效性判断**：设定阈值，低于阈值提示重新采集

### 4.2 算法优化要求

#### 4.2.1 防过拟合优化
**技术措施**：
- **数据增强**：通过数据增强技术扩充训练样本
- **正则化**：L1/L2正则化、Dropout等技术
- **交叉验证**：K折交叉验证确保模型泛化能力
- **早停策略**：监控验证集性能，防止过拟合

**模型评估指标**：
- **准确率**：体质识别准确率（目标：≥85%）
- **精确率与召回率**：各类体质的精确率与召回率
- **F1分数**：综合评估模型性能
- **AUC值**：ROC曲线下面积（二分类问题）

#### 4.2.2 模型可解释性
- **特征重要性分析**：展示哪些HRV参数对体质判断影响最大
- **决策路径可视化**：展示模型判断过程
- **置信度评估**：提供预测结果的置信度分数

### 4.3 技术架构建议
- **前端**：小程序框架（微信小程序/uni-app等）
- **后端**：云服务架构，支持数据存储与算法计算
- **算法服务**：独立的算法服务模块，支持模型更新与优化
- **数据安全**：用户数据加密存储，符合《个人信息保护法》要求
