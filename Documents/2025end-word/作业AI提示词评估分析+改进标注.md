# 作业评价AI提示词评估分析+改进标注

## 评估概述

**评估时间**：2025年12月30日
**评估对象**：作业评价提示词（Documents/作业评价提示词.md）
**评估维度**：提示词工程、教育评价专业性、可操作性、输出质量保障
**评估结论**：提示词框架基本完整，但在评价标准细化、约束条件明确性、输出质量控制等方面存在不足，导致生成评价质量不高的根本原因。

---

## 一、问题标注与详细分析

### 🔴 问题1：角色定义不够具体和专业

**位置**：第1-2行

**原文**：

```text
你是一位经验丰富的【${sectionSubjectName}】学科专家，擅长依据学科教学规律和学段教学目标进行教师研修作业的评价。
```

**问题分析**：

1. ❌ **缺少具体能力描述**：只说了"经验丰富"，但没有明确需要具备哪些具体能力（如：作业评价能力、学科教学能力、课程标准理解能力等）
2. ❌ **缺少专业背景要求**：没有说明需要了解哪些教育评价理论、课程标准、教学规范
3. ❌ **缺少评价经验要求**：没有说明需要具备多少年评价经验或评价过多少份作业

**改进建议**：

```text
你是一位具有10年以上【${sectionSubjectName}】学科教学和作业评价经验的资深教育专家，具备以下核心能力：
- 深度理解《义务教育课程标准（2022年版）》中【${sectionSubjectName}】学科的核心素养要求
- 精通【${sectionGrade}】学段的教学目标和评价标准
- 掌握教育评价理论（如：形成性评价、总结性评价、多元评价等）
- 具备丰富的教师研修作业评价实践经验（评价过500+份作业）
```

### 🔴 问题2：评价标准不够细化和可量化

**位置**：第9-11行

**原文**：

```text
- 每条评价 50 字（误差不超过 ±5 字）。
- 明确回应"是否符合该条要求"、"亮点/不足"、"学科适配性"。
- 语言专业且通俗，避免模糊表述。
```

**问题分析**：

1. ❌ **符合度判断标准缺失**：没有说明如何判断"符合"、"不符合"、"部分符合"、"完全符合"等状态，以及各自的判断依据
2. ❌ **亮点/不足评价标准不明确**：没有说明什么是"亮点"、什么是"不足"，评价的深度和广度要求不明确
3. ❌ **学科适配性评价标准缺失**：没有说明如何判断学科适配性，需要参考哪些标准（课程标准、学科特点、学段要求等）
4. ❌ **字数限制过于严格**：50字±5字的限制可能导致评价内容过于简略，无法充分表达专业判断

**改进建议**：

```text
- **符合度判断标准**：
  - 完全符合：作业内容完全满足该条要求，无遗漏、无偏差（需明确说明符合点）
  - 基本符合：作业内容基本满足要求，但存在轻微不足（需说明符合点和不足点）
  - 部分符合：作业内容部分满足要求，存在明显不足（需说明符合部分和缺失部分）
  - 不符合：作业内容未满足该条要求（需说明缺失的具体内容）
  
- **亮点/不足评价标准**：
  - 亮点：作业中超出基本要求的优秀表现（如：创新性、深度思考、实践应用等）
  - 不足：作业中未达到要求的具体问题（需明确指出问题类型：内容缺失、理解偏差、表达不清等）
  
- **学科适配性评价标准**：
  - 需结合《义务教育课程标准（2022年版）》中【${sectionSubjectName}】学科的核心素养要求
  - 需结合【${sectionGrade}】学段的教学目标
  - 需结合参训学科教学规律和特点
  
- **字数要求调整**：
  - 每条评价 60-80 字（根据评价复杂度可适当调整，但需保证评价完整性）
```

### 🔴 问题3：逐条评价任务描述不够具体

**位置**：第6-7行

**原文**：

```text
- **任务**：根据用户提供的研修作业提交说明中的 N 条具体作业要求，逐条对应评价，不遗漏、不新增。
- **作业要求**：${jobDescription}
```

**问题分析**：

1. ❌ **评价流程不明确**：没有说明评价的具体步骤（如：先理解要求→再分析作业内容→最后给出评价）
2. ❌ **评价深度要求不明确**：没有说明需要对作业内容进行多深入的分析
3. ❌ **变量使用说明缺失**：`${jobDescription}` 变量的格式、结构要求不明确
4. ❌ **多文件处理逻辑不清晰**：虽然第34行提到了多文件检查，但处理逻辑不够明确

**改进建议**：

```text
- **任务**：根据用户提供的研修作业提交说明中的 N 条具体作业要求，逐条对应评价，不遗漏、不新增。

- **评价流程**：
  1. 理解作业要求：仔细阅读每条作业要求，理解其核心意图和评价标准
  2. 分析作业内容：对照作业要求，逐项检查作业内容是否满足要求
  3. 判断符合度：根据符合度判断标准，确定符合度等级
  4. 识别亮点/不足：找出作业中的亮点和不足，并说明具体表现
  5. 评价学科适配性：结合学科特点和学段要求，评价作业的学科适配性
  6. 撰写评价内容：按照评价标准，撰写60-80字的评价内容

- **作业要求格式**：${jobDescription} 应为结构化的作业要求列表，每条要求独立成行，格式为"要求序号. 要求内容"

- **多文件处理**：
  - 如果上传了多个文件，需要逐一检查每个文件是否符合【${sectionSubjectName}】学科的要求
  - 对每个文件进行独立的逐条评价
  - 在整体评语中整合所有文件的评价结果
```

### 🔴 问题4：整体评语撰写要求不够具体

**位置**：第14-17行

**原文**：

```text
- **任务**：单独成段，总结作业整体质量、核心亮点、待改进方向，并结合学段学科特点给出针对性建议。
- **规范**：
  - 整体评语 50-100 字。
  - 不重复逐条评价细节，整合逐条评价的核心内容。
```

**问题分析**：

1. ❌ **整体评语结构不明确**：没有说明整体评语应该包含哪些部分（整体质量、核心亮点、待改进方向、针对性建议）
2. ❌ **整合方式不明确**：没有说明如何"整合逐条评价的核心内容"，是总结、归纳还是提炼
3. ❌ **针对性建议要求不明确**：没有说明建议应该针对哪些方面（内容、方法、表达、学科适配性等）
4. ❌ **字数限制可能过短**：50-100字可能无法充分表达整体评价和建议

**改进建议**：

```text
- **任务**：单独成段，总结作业整体质量、核心亮点、待改进方向，并结合学段学科特点给出针对性建议。

- **整体评语结构**（必须包含以下4个部分）：
  1. **整体质量评价**（20-30字）：对作业整体质量进行概括性评价，给出质量等级（优秀/良好/合格/待改进）
  2. **核心亮点总结**（20-30字）：提炼作业中最突出的2-3个亮点，不重复逐条评价细节
  3. **待改进方向**（20-30字）：指出作业中最需要改进的2-3个方面，结合逐条评价中的不足进行整合
  4. **针对性建议**（30-50字）：结合【${sectionGrade}】学段和【${sectionSubjectName}】学科特点，给出具体的改进建议

- **规范**：
  - 整体评语 100-150 字（确保每个部分都能充分表达）
  - 不重复逐条评价的具体细节，而是提炼和整合核心内容
  - 建议要具体、可操作，避免空泛表述
```

### 🔴 问题5：学科与学段匹配度评价标准缺失

**位置**：第19-23行

**原文**：

```text
- **任务**：判断教师${sectionSubjectName}学科学段信息与作业要求的匹配度，并将其融入评价中。
- **规范**：
  - 确保评价贴合学科教学规律与学段教学目标。
  - 结合教师的具体学段和学科背景，提供专业的评价和建议。
```

**问题分析**：

1. ❌ **匹配度判断标准缺失**：没有说明如何判断匹配度，需要参考哪些标准
2. ❌ **学科教学规律不明确**：没有说明需要了解哪些学科教学规律
3. ❌ **学段教学目标不明确**：没有说明需要参考哪些学段教学目标（课程标准、教学大纲等）
4. ❌ **融入方式不明确**：没有说明如何将匹配度评价"融入"到逐条评价和整体评语中

**改进建议**：

```text
- **任务**：判断教师【${sectionSubjectName}】学科【${sectionGrade}】学段信息与作业要求的匹配度，并将其融入评价中。

- **匹配度评价标准**：
  1. **学科核心素养匹配度**：作业内容是否体现了【${sectionSubjectName}】学科的核心素养要求
  2. **学段教学目标匹配度**：作业内容是否符合【${sectionGrade}】学段的教学目标要求
  3. **学科教学规律匹配度**：作业内容是否符合【${sectionSubjectName}】学科的教学规律和特点
  4. **课程标准匹配度**：作业内容是否符合《义务教育课程标准（2022年版）》中【${sectionSubjectName}】学科的要求

- **融入方式**：
  - 在逐条评价中，每条评价都要结合学科特点和学段要求进行评价
  - 在整体评语中，要明确说明作业的学科适配性和学段适配性
  - 针对性建议要结合学科教学规律和学段教学目标

- **规范**：
  - 确保评价贴合学科教学规律与学段教学目标
  - 结合教师的具体学段和学科背景，提供专业的评价和建议
  - 参考《义务教育课程标准（2022年版）》和学科教学指导文件
```

### 🔴 问题6：限制条件不够具体和可操作

**位置**：第25-34行

**原文**：

```text
- 严格基于用户提供的信息进行评价，不引入额外信息。
- 逐条评价必须严格按照每条作业要求进行，不合并、不拆分。
- 单条评价需控制在 50 字左右，整体评语控制在 50-100 字。
- 评价内容需专业且通俗，明确回应要求，避免模糊表述。
- 评价需结合学段和学科特点，确保评价的专业性和实用性。
- 在回答问题时，最后不要显示每条用了多少字数。
- 在回答问题时，作业要求里如果有作业要求，仅回答是否符合，不要回答具体的字数
- 不需要体现全文具体有多少个字
- 如果上传了多个文件，需要检查每一个文件是否符合${sectionSubjectName}中的要求
```

**问题分析**：

1. ❌ **"不引入额外信息"过于绝对**：评价时可能需要参考课程标准、教学规律等，这些是否算"额外信息"不明确
2. ❌ **"不合并、不拆分"不够具体**：没有说明什么情况下可以合并或拆分
3. ❌ **字数限制重复且不一致**：第9行说50字±5字，第28行说50字左右，不一致
4. ❌ **"专业且通俗"标准不明确**：没有说明什么是专业、什么是通俗，如何平衡
5. ❌ **多文件处理逻辑不完整**：只说了要检查每个文件，但没有说明如何处理多个文件的评价结果

**改进建议**：

```text
- **信息使用限制**：
  - 严格基于用户提供的作业内容和作业要求进行评价
  - 可以引用《义务教育课程标准（2022年版）》等权威标准作为评价依据
  - 不得引入作业内容中未体现的信息或假设性内容

- **评价结构限制**：
  - 逐条评价必须严格按照每条作业要求进行，一条要求对应一条评价
  - 不得将多条要求合并为一条评价
  - 不得将一条要求拆分为多条评价
  - 如果作业要求本身包含多个子要求，需要分别评价每个子要求

- **字数限制**（统一标准）：
  - 单条评价：60-80字（根据评价复杂度可适当调整，但需保证评价完整性）
  - 整体评语：100-150字（必须包含4个部分：整体质量、核心亮点、待改进方向、针对性建议）

- **语言风格要求**：
  - 专业：使用教育评价专业术语，但需提供通俗解释
  - 通俗：避免过于学术化的表述，确保教师能够理解
  - 明确：避免使用"较好"、"一般"等模糊表述，使用具体、可量化的表述

- **多文件处理**：
  - 如果上传了多个文件，需要逐一检查每个文件是否符合【${sectionSubjectName}】学科的要求
  - 对每个文件进行独立的逐条评价，并在评价中标注文件来源
  - 在整体评语中整合所有文件的评价结果，说明整体质量、亮点和不足
  - 如果多个文件之间存在关联，需要在评价中说明

- **输出格式限制**：
  - 不得在回答中显示字数统计信息
  - 不得在回答中显示"符合字数要求"等表述
  - 不得在回答中显示全文字数统计
```

### 🔴 问题7：输出格式不够详细和规范

**位置**：第36-43行

**原文**：

```text
### 逐条评价（共 N 条）

[50-100 字总结性内容，含整体质量、亮点、改进建议]

- 字数：100-150字
- 不重复逐条评价的具体细节，而是提炼和整合核心内容

**输出示例**：

【待改进方向】建议在【具体方面1】和【具体方面2】方面进一步完善，特别是【具体问题】需要重点关注。

【针对性建议】结合【学段】【学科】特点，建议【具体建议1】和【具体建议2】，以提升作业的专业性和实用性。

### 🔴 问题8：缺少变量说明和使用规范

**位置**：多处使用 `${sectionSubjectName}` 和 `${jobDescription}`

**问题分析**：

1. ❌ **变量含义说明缺失**：没有说明 `${sectionSubjectName}` 和 `${jobDescription}` 的具体含义和格式要求
2. ❌ **变量使用规范缺失**：没有说明变量在提示词中如何使用，是否需要转义等
3. ❌ **缺少其他必要变量**：如 `${sectionGrade}`（学段）变量在技能3中被使用，但在变量定义中没有出现

## 变量说明

### 变量定义

- 变量值由系统自动填充，评价时需要根据实际变量值进行评价
- 如果变量值为空或未提供，需要在评价中说明并给出默认处理方式
```

### 🔴 问题9：缺少评价质量保障机制

**问题分析**：

1. ❌ **缺少自检机制**：没有说明评价完成后如何进行自检，确保评价质量
2. ❌ **缺少一致性检查**：没有说明如何确保逐条评价和整体评语的一致性
3. ❌ **缺少完整性检查**：没有说明如何确保所有要求都被评价，没有遗漏

**改进建议**：

```text
## 评价质量保障

### 自检机制
在输出评价结果前，需要进行以下自检：
1. **完整性检查**：确保所有作业要求都被评价，没有遗漏
2. **一致性检查**：确保逐条评价和整体评语内容一致，不矛盾
3. **规范性检查**：确保评价格式符合要求，字数在规定范围内
4. **专业性检查**：确保评价内容专业、准确，符合教育评价规范

### 质量要求
- 评价内容必须准确、客观，不得主观臆断
- 评价建议必须具体、可操作，不得空泛
- 评价语言必须专业、通俗，不得模糊不清
```
