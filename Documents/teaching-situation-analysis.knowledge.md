# 教情分析数据接入与处理规范

## 教情分析数据来源

### 数据文件

- **文件路径**：`888-PromptX/knowledgebase/教情分析表.pdf`
- **数据类型**：教学情况分析报告
- **更新频率**：按学期/学年更新
- **数据内容**：知识维度、技能维度、能力素养维度的长板/短板分析、偏态量、策略建议

### 核心数据项

| 数据项       | 说明                     | 数据格式  |
| ------------ | ------------------------ | --------- |
| 知识维度长板 | 教师在知识维度的强项     | 文本描述  |
| 知识维度短板 | 教师在知识维度的弱项     | 文本描述  |
| 技能维度长板 | 教师在技能维度的强项     | 文本描述  |
| 技能维度短板 | 教师在技能维度的弱项     | 文本描述  |
| 能力素养长板 | 教师在能力素养维度的强项 | 文本描述  |
| 能力素养短板 | 教师在能力素养维度的弱项 | 文本描述  |
| 偏态量       | 各维度偏态分析（+/-）    | 数值+符号 |
| 策略建议     | 3针对性改进策略          | 文本描述  |

## 数据接入流程

### A. 教情分析数据接入

- **A1 文件解析**：解析 `教情分析表.pdf`中的结构化数据
- **A2 数据提取**：提取知识、技能、能力素养三个维度的长板/短板信息
- **A3 格式校验**：验证长板/短板标识、偏态量（+/-）、策略建议文本的完整性
- **A4 数据关联**：将教情分析数据与教师培训画像的对应维度关联

### B. 数据处理规则

- **B1 长板识别**：识别教师在各维度的长板，用于培训方向优化
- **B2 短板识别**：识别教师在各维度的短板，用于培训需求匹配
- **B3 偏态分析**：基于偏态量分析，提供针对性培训建议
- **B4 策略转化**：将教情分析的策略建议转化为具体的培训方向和提升重点

## 教情分析对决策的策略和规则

### 智能决策分析模块增强

#### （2）【智能决策分析模块】教情分析对决策的策略和规则

**策略1：长板强化策略**

- **触发条件**：教情分析显示某维度存在明显长板
- **决策规则**：推荐强化该长板的高级培训课程
- **实施方式**：提供进阶培训、专业发展、导师角色等建议

**策略2：短板补强策略**

- **触发条件**：教情分析显示某维度存在明显短板
- **决策规则**：优先推荐补强该短板的基础培训课程
- **实施方式**：提供基础培训、技能提升、同伴互助等建议

**策略3：偏态校正策略**

- **触发条件**：教情分析偏态量显示某维度偏态明显（+/-）
- **决策规则**：提供平衡发展的培训规划
- **实施方式**：推荐平衡性培训、跨领域学习、综合能力提升等

**策略4：策略融合规则**

- **触发条件**：教情分析提供具体策略建议
- **决策规则**：将策略建议与现有培训推荐模型融合
- **实施方式**：生成融合教情分析建议的个性化培训方案

### 教情分析数据应用规则

**规则1：数据优先级**

- 教情分析数据在智能决策中具有高优先级
- 与其他数据源冲突时，以教情分析为准

**规则2：关联分析**

- 将教情分析的短板与培训需求关联
- 将教情分析的长板与培训方向优化关联

**规则3：动态更新**

- 教情分析数据更新时，自动重新生成相关推荐
- 保持培训建议与最新教情分析结果的一致性

## 数据质量要求

### 完整性要求

- 教情分析数据必须包含三个维度（知识、技能、能力素养）的长板/短板信息
- 必须包含偏态量分析结果
- 必须包含具体的策略建议

### 一致性要求

- 教情分析数据应与教师其他维度数据保持逻辑一致性
- 避免出现矛盾的分析结论

## 异常处理

### 数据缺失处理

- **IF** 教情分析数据缺失 **THEN** 显示"暂无教情分析数据，接入后可提供更精准的培训建议"

### 数据冲突处理

- **IF** 教情分析与其他数据源冲突 **THEN** 优先采用教情分析结果并标注数据来源

### 格式错误处理

- **IF** 教情分析数据格式错误 **THEN** 生成格式修正提示清单
