# 培训数据清洗研发需求2.0

## 1. 需求背景

在教师培训画像系统中，数据质量直接影响分析结果的准确性和推荐的有效性。由于数据来源多样（教师信息管理系统、培训管理系统、教情分析表、数字素养测评系统等），格式不统一，存在缺失值、异常值、重复数据等问题，需要建立一套完整的、智能化的数据清洗机制，确保输入到系统的数据质量达到分析要求。同时，需确保数据清洗过程符合《国培计划课程标准》《小学/中学教师专业标准（试行）》等政策要求。

## 2. 产品目标

- 建立标准化、智能化的数据清洗流程
- 提高数据质量，减少错误分析结果
- 实现自动化数据清洗，减少人工干预
- 提供数据质量监控和异常处理机制
- 确保数据处理符合教育行业政策标准
- 支持多源异构数据的统一处理

## 3. 功能需求

### 3.1 数据接入
- **多源数据接入**：接入来自教师信息管理系统、培训管理系统、教情分析表、数字素养测评系统、AI课堂观察系统等多源数据
- **批量数据接入**：支持批量导入CSV、Excel、JSON等格式的培训数据
- **实时数据接入**：支持从API接口实时获取培训相关数据
- **文件格式接入**：支持PDF格式的教情分析表数据接入，提取结构化信息
- **标准数据接入**：接入《国培计划课程标准》《小学/中学教师专业标准（试行）》等政策标准数据
- **历史数据接入**：接入教师历史培训记录、能力评估数据、发展轨迹数据

### 3.2 数据清洗

#### 3.2.1 格式标准化清洗
- **时间格式标准化**：
  - 输入格式：支持"2025.2"、"2025/2"、"2025-2"、"2025年2月"、"Feb 2025"等多种格式
  - 输出格式：统一转换为"YYYY年MM月"格式
  - 验证规则：确保月份在1-12范围内，年份在合理范围内

- **趋势标识标准化**：
  - 输入格式：支持"上升"、"下降"、"持平"、"↑"、"↓"、"-"、"up"、"down"、"stable"等
  - 输出格式：统一转换为"↑"、"↓"、"-"标准符号
  - 验证规则：确保趋势标识与数据变化方向一致

- **教龄格式转换**：
  - 输入格式：支持"四年"、"四"、"肆"、"4年"、"4"、"four"等
  - 输出格式：统一转换为阿拉伯数字（如：4）
  - 验证规则：确保教龄为正整数，不超过合理范围（如≤50年）

- **分数范围验证**：
  - 输入格式：支持各种分数表示（如85、85.5、85.0等）
  - 输出格式：统一为0-100分范围内的数值
  - 验证规则：IF 分数<0 OR 分数>100 THEN 提示修正并记录异常

- **培训学时标准化**：
  - 输入格式：支持各种学时表示（如8、8.0、8.5等）
  - 输出格式：统一为0.5的倍数
  - 验证规则：IF 学时%0.5≠0 THEN 提示修正

- **学科分类标准化**：
  - 输入格式：支持各种学科表述（如"数学"、"Math"、"mathematics"、"小学数学"等）
  - 输出格式：统一为标准学科分类代码
  - 验证规则：确保学科在标准分类体系内

#### 3.2.2 缺失值处理
- **必填字段识别**：识别姓名、教龄、学科等必填字段
- **缺失值分类**：
  - 完全缺失：字段值为空或null
  - 部分缺失：字段值不完整（如只有姓没有名）
  - 逻辑缺失：字段值存在但不符合业务逻辑
- **缺失值处理策略**：
  - IF 姓名缺失 THEN 标记为严重错误，需人工处理
  - IF 教龄缺失 THEN 根据入职时间推算（IF 入职时间存在）
  - IF 其他字段缺失 THEN 标记为"当前暂无XX数据，补充后可自动更新"

#### 3.2.3 异常值检测
- **逻辑冲突检测**：
  - IF 培训时间早于入职时间 THEN 标记为异常
  - IF 教龄大于工作年限 THEN 标记为异常
  - IF 培训学时为负数 THEN 标记为异常

- **范围异常检测**：
  - IF 分数不在0-100范围内 THEN 标记为异常
  - IF 教龄不在0-50范围内 THEN 标记为异常
  - IF 培训级别不在预定义范围内 THEN 标记为异常

- **关联缺失检测**：
  - IF 培训记录缺少对应的证书状态 THEN 标记为待补充
  - IF 教情分析缺少偏态量 THEN 标记为不完整

#### 3.2.4 重复数据处理
- **教师信息去重**：
  - 基于姓名+身份证号+学科+入职时间的组合判断
  - 保留最新或最完整的记录
  - 记录去重过程日志

- **培训记录去重**：
  - 基于教师ID+培训项目名称+培训时间的组合判断
  - 保留已获得证书的记录
  - 记录去重过程日志

- **评估数据去重**：
  - 基于教师ID+评估时间+评估维度的组合判断
  - 保留分数最高的记录
  - 记录去重过程日志

#### 3.2.5 敏感信息脱敏
- **姓名脱敏**：保留姓，名替换为*
- **身份证号脱敏**：保留前6位和后4位，中间用*代替
- **联系方式脱敏**：保留区号，号码中间用*代替
- **符合法规要求**：确保脱敏后数据符合《个人信息保护法》和《数据安全法》要求

### 3.4 产品校验
- **清洗效果校验**：验证数据清洗后的质量提升效果，对比清洗前后数据质量指标
- **格式合规性校验**：确保清洗后的数据符合预设格式要求和政策标准
- **完整性校验**：验证关键字段的完整性，确保必填字段不缺失
- **逻辑一致性校验**：确保清洗后的数据在逻辑上保持一致
- **业务规则校验**：验证清洗后的数据符合业务规则要求
- **隐私保护校验**：确保敏感信息已正确脱敏
- **政策合规性校验**：验证数据处理符合《国培计划课程标准》等政策要求
- **性能校验**：验证清洗处理的效率和资源消耗

### 3.5 功能规则

#### 3.5.1 数据校验规则
- **A1 数据完整性校验规则**：
  - 自动检测各工作表的必填字段是否完整
  - IF 姓名字段为空 THEN 标记为严重错误
  - IF 教龄字段为空 THEN 根据入职时间推算（IF 入职时间存在）
  - IF 学科字段为空 THEN 标记为待补充

- **A2 数据格式校验规则**：
  - IF 时间格式不符合"YYYY年MM月" THEN 自动转换或提示修正
  - IF 趋势标识不是"↑/↓/-" THEN 转换为标准格式
  - IF 教龄不是数字格式 THEN 进行格式转换
  - IF 分数不在0-100范围内 THEN 提示修正
  - IF 培训学时不是0.5的倍数 THEN 提示修正

#### 3.5.2 数据清洗规则
- **B1 格式标准化规则**：
  - 自动执行时间格式、趋势标识、教龄格式、分数范围、学时单位的标准化
  - 对常见错误格式进行智能匹配和转换
  - IF 无法自动转换 THEN 记录异常并提示人工处理

- **B2 缺失值处理规则**：
  - IF 某模块数据不足 THEN 在报告对应区域显示"当前暂无XX数据，补充后可自动更新"
  - IF 必填字段缺失 THEN 标记为严重错误，需人工干预
  - IF 非必填字段缺失 THEN 保持缺失状态但记录日志

- **B3 异常值处理规则**：
  - IF 能力维度分数与常模/区域值差异过大（超过30分） THEN 增加"该维度为当前薄弱项，建议优先参与对应培训"的标注
  - IF 逻辑冲突数据 THEN 标记为异常并提示修正
  - IF 范围异常数据 THEN 提示修正或删除

- **B4 重复数据处理规则**：
  - 自动识别并合并重复的教师信息
  - 去除完全重复的培训记录
  - 保留最新或最完整的记录

#### 3.5.3 隐私保护规则
- **C1 敏感信息处理规则**：对教师姓名、身份证号、联系方式等敏感信息自动做模糊化处理
- **C2 合规性校验规则**：确保数据处理符合《个人信息保护法》《数据安全法》《教育数据安全管理办法》
- **C3 数据最小化规则**：仅处理分析所需的最小数据集

#### 3.5.4 质量控制规则
- **D1 质量阈值规则**：
  - IF 数据完整性低于80% THEN 标记为低质量数据集
  - IF 异常数据比例超过5% THEN 触发质量告警
  - IF 重复数据比例超过3% THEN 触发质量告警

- **D2 清洗日志规则**：记录数据清洗过程，包括操作类型、处理数据量、处理时间、处理结果等，保证可追溯性

- **D3 验证反馈规则**：清洗后验证数据质量，不符合要求的返回重新处理，并生成质量报告

#### 3.5.5 政策标准匹配规则
- **E1 国培计划标准匹配规则**：
  - IF 教师数据符合国培计划培训对象要求 THEN 标记为符合条件
  - IF 数据格式符合国培计划标准 THEN 通过格式校验

- **E2 教师专业标准匹配规则**：
  - IF 数据字段符合《小学/中学教师专业标准（试行）》要求 THEN 通过标准校验
  - IF 评估维度符合专业标准 THEN 通过维度校验

#### 3.5.6 自动化处理规则
- **F1 自动清洗规则**：
  - 对于明确的格式错误，自动进行转换
  - 对于明显的重复数据，自动进行去重
  - 对于确定的异常值，自动进行修正

- **F2 人工干预规则**：
  - 对于复杂的逻辑错误，标记待人工处理
  - 对于敏感信息的处理，需要确认后执行
  - 对于重要的数据修正，需要人工确认

## 4. 技术要求

- 支持多种数据格式的批量处理，包括CSV、Excel、JSON、XML、PDF等
- 实现实时数据清洗和批量数据清洗，支持流式处理
- 提供数据质量监控仪表板，实时显示数据质量指标
- 确保数据处理过程的可追溯性，记录完整的处理日志
- 支持数据清洗规则的动态配置和热更新
- 实现高并发处理能力，支持大数据量处理
- 提供API接口，支持与其他系统集成
- 确保数据安全，实现端到端加密

- 标准化的数据格式和结构，符合教育行业标准
- 高质量的清洗后数据，准确性达到95%以上
- 自动化的数据清洗流程，人工干预率低于5%
- 完善的数据质量监控机制，实时监控数据质量
- 符合隐私保护要求的数据处理，通过合规性检查
- 政策标准符合的数据集，满足国培计划要求
- 可追溯的数据处理过程，支持审计需求
