# 培训数据清洗研发需求

## 1. 需求背景

在教师培训画像系统中，数据质量直接影响分析结果的准确性和推荐的有效性。由于数据来源多样，格式不统一，存在缺失值、异常值、重复数据等问题，需要建立一套完整的数据清洗机制，确保输入到系统的数据质量达到分析要求。

## 2. 产品目标

- 建立标准化的数据清洗流程
- 提高数据质量，减少错误分析结果
- 实现自动化数据清洗，减少人工干预
- 提供数据质量监控和异常处理机制

## 3. 功能需求

### 3.1 数据接入
- **多源数据接入**：接入来自教师信息管理系统、培训管理系统、教情分析表、数字素养测评系统等多源数据
- **批量数据接入**：支持批量导入CSV、Excel等格式的培训数据
- **实时数据接入**：支持从API接口实时获取培训相关数据
- **文件格式接入**：支持PDF格式的教情分析表数据接入

### 3.2 数据清洗
- **格式标准化清洗**：
  - 时间格式统一：将"2025.2"、"2025/2"、"2025-2"等格式统一转换为"YYYY年MM月"
  - 趋势标识标准化：将"上升"、"下降"、"持平"等文本统一转换为"↑"、"↓"、"-"
  - 教龄格式转换：将"四年"、"四"、"肆"等中文数字转换为阿拉伯数字
  - 分数范围验证：确保分数在0-100分范围内，超出范围的数据进行提示修正
  - 培训学时标准化：确保学时为0.5的倍数，去除0.2、0.1等不合规数值

- **缺失值处理**：
  - 识别必填字段缺失情况
  - 对缺失数据进行标识和分类
  - 提供缺失数据补充建议

- **异常值检测**：
  - 识别逻辑冲突数据（如培训时间早于入职时间）
  - 检测范围异常数据（如分数超出合理范围）
  - 发现关联缺失数据（如缺少必要的关联信息）

- **重复数据处理**：
  - 识别重复的教师信息记录
  - 去除重复的培训项目记录
  - 处理相似但不完全重复的数据

- **敏感信息脱敏**：
  - 对教师姓名等敏感信息进行模糊化处理
  - 保护教师隐私数据
  - 符合《个人信息保护法》和《数据安全法》要求

### 3.3 AI提示词工程实现机制
- **数据质量评估提示词**：评估接入数据的整体质量，识别主要问题
- **清洗规则应用提示词**：根据数据类型和问题类型，应用相应的清洗规则
- **异常处理提示词**：生成异常数据的处理建议和用户提示
- **质量报告生成提示词**：生成数据清洗质量报告，包括清洗前后对比

### 3.4 产品校验
- **清洗效果校验**：验证数据清洗后的质量提升效果
- **格式合规性校验**：确保清洗后的数据符合预设格式要求
- **完整性校验**：验证关键字段的完整性
- **逻辑一致性校验**：确保清洗后的数据在逻辑上保持一致
- **业务规则校验**：验证清洗后的数据符合业务规则要求

### 3.5 功能规则
- **数据校验规则**：
  - **A1 数据校验规则**：自动检测各工作表的必填字段是否完整、数据格式是否合规
  - **A2 数据格式规则**：
    - IF 时间格式不符合"YYYY年MM月" THEN 自动转换或提示修正
    - IF 趋势标识不是"↑/↓/-" THEN 转换为标准格式
    - IF 教龄不是数字格式 THEN 进行格式转换
    - IF 分数不在0-100范围内 THEN 提示修正
    - IF 培训学时不是0.5的倍数 THEN 提示修正

- **数据清洗规则**：
  - **B1 格式标准化规则**：
    - 自动执行时间格式、趋势标识、教龄格式、分数范围、学时单位的标准化
    - 对常见错误格式进行智能匹配和转换
  - **B2 缺失值处理规则**：
    - IF 某模块数据不足 THEN 在报告对应区域显示"当前暂无XX数据，补充后可自动更新"
  - **B3 异常值处理规则**：
    - IF 能力维度分数与常模/区域值差异过大 THEN 增加"该维度为当前薄弱项，建议优先参与对应培训"的标注
  - **B4 重复数据处理规则**：
    - 自动识别并合并重复的教师信息
    - 去除完全重复的培训记录

- **隐私保护规则**：
  - **C1 敏感信息处理规则**：对教师敏感信息自动做模糊化处理
  - **C2 合规性校验规则**：确保数据处理符合《个人信息保护法》《数据安全法》《教育数据安全管理办法》

- **质量控制规则**：
  - **D1 质量阈值规则**：设定数据质量阈值，低于阈值的数据拒绝处理
  - **D2 清洗日志规则**：记录数据清洗过程，保证可追溯性
  - **D3 验证反馈规则**：清洗后验证数据质量，不符合要求的返回重新处理

## 4. 技术要求

- 支持多种数据格式的批量处理
- 实现实时数据清洗和批量数据清洗
- 提供数据质量监控仪表板
- 确保数据处理过程的可追溯性
- 支持数据清洗规则的动态配置

## 5. 预期成果

- 标准化的数据格式和结构
- 高质量的清洗后数据
- 自动化的数据清洗流程
- 完善的数据质量监控机制
- 符合隐私保护要求的数据处理
