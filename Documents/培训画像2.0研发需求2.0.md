# 培训画像2.0研发需求2.0

## 1. 需求背景

教师培训画像系统是帮助教育管理者和教师本人了解培训需求、评估培训效果的重要工具。当前的培训画像系统需要升级到2.0版本，以提供更精准、更全面的培训分析和建议。新版本将整合更多数据源，特别是教情分析数据，并结合《国培计划课程标准》《小学/中学教师专业标准（试行）》等政策标准，实现更深入的洞察和更智能的决策。

## 2. 产品目标

- 实现基于国家标准的教师培训需求精准分析
- 整合教情分析数据，提供教学实际情况洞察
- 优化智能决策分析，提供个性化培训建议
- 提升用户体验和报告可视化效果
- 生成标准化的教师数字画像分析报告

## 3. 功能需求

### 3.1 数据接入
- **教师基础信息接入**：接入教师姓名、教龄、地区、学科、职务、入职时间等基础信息
- **专业发展数据接入**：接入专业理念与师德、专业知识、专业技能、专业发展、数字素养等维度的评估数据
- **教情分析数据接入**：接入知识维度、技能维度、能力素养维度的长板/短板分析、偏态量、策略建议（来自knowledgebase/教情分析表.pdf）
- **培训历史数据接入**：接入教师过往培训项目、级别、时间、学时、证书状态、结业状态等信息
- **数字素养数据接入**：接入教育技术应用、在线教学、数字化资源开发、数据驱动教学改进等能力数据
- **教研能力数据接入**：接入AI课堂观察、微格教学评价、学业质量与能力素养评价等评估数据
- **政策标准数据接入**：接入《国培计划课程标准》《小学/中学教师专业标准（试行）》等政策标准数据
- **区域对比数据接入**：接入区域平均值、TeachSeek常模值等对比基准数据

### 3.2 数据清洗
- **数据格式标准化**：
  - 时间格式统一：将"2025.2"、"2025/2"、"2025-2"、"2025年2月"等格式统一转换为"YYYY年MM月"
  - 趋势标识标准化：将"上升"、"下降"、"持平"等文本统一转换为"↑"、"↓"、"-"
  - 教龄格式转换：将"四年"、"四"、"肆"等中文数字转换为阿拉伯数字
  - 分数范围验证：确保分数在0-100分范围内，超出范围的数据进行提示修正
  - 培训学时标准化：确保学时为0.5的倍数，去除0.2、0.1等不合规数值
  - 学科分类标准化：将不同表述的学科名称统一为标准分类

- **缺失值处理**：
  - 识别必填字段缺失情况（如姓名、教龄、学科等）
  - 对缺失数据进行标识，显示"当前暂无XX数据，补充后可自动更新"
  - 提供缺失数据补充建议

- **异常值检测**：
  - 识别逻辑冲突数据（如培训时间早于入职时间）
  - 检测范围异常数据（如分数超出合理范围）
  - 发现关联缺失数据（如缺少必要的关联信息）

- **重复数据处理**：
  - 识别重复的教师信息记录
  - 去除重复的培训项目记录
  - 处理相似但不完全重复的数据

- **敏感信息脱敏**：
  - 对教师姓名等敏感信息进行模糊化处理
  - 保护教师隐私数据
  - 符合《个人信息保护法》和《数据安全法》要求

### 3.4 产品校验
- **数据完整性校验**：验证各数据维度的完整性
- **逻辑一致性校验**：确保分析结果与输入数据的逻辑一致性
- **推荐合理性校验**：验证培训推荐与教师实际情况的匹配度
- **可视化准确性校验**：确保图表数据与分析结果的一致性
- **异常处理校验**：验证异常数据的处理和提示机制
- **政策合规性校验**：验证分析结果符合《国培计划课程标准》等政策要求
- **教情分析有效性校验**：验证教情分析数据的应用效果

### 3.5 功能规则

#### 3.5.1 教情分析应用规则
- **长板强化策略**：
  - IF 教情分析显示某维度存在明显长板（分数高于区域平均值20分以上）THEN 推荐强化该长板的高级培训课程
  - 推荐方向：进阶培训、专业发展、导师角色等
  - 优先级：中等优先级
  - 应用场景：教师在某方面表现突出，可进一步发展为专家型教师

- **短板补强策略**：
  - IF 教情分析显示某维度存在明显短板（分数低于区域平均值15分以上）THEN 优先推荐补强该短板的基础培训课程
  - 推荐方向：基础培训、技能提升、同伴互助等
  - 优先级：高优先级
  - 应用场景：教师在某方面存在明显不足，急需提升

- **偏态校正策略**：
  - IF 教情分析偏态量显示某维度偏态明显（+/-值超过0.5）THEN 提供平衡发展的培训规划
  - 推荐方向：平衡性培训、跨领域学习、综合能力提升等
  - 优先级：根据偏态程度确定
  - 应用场景：教师在某维度发展不均衡，需要调整

- **策略融合规则**：
  - IF 教情分析提供具体策略建议 THEN 将策略建议与现有培训推荐模型融合
  - 生成融合教情分析建议的个性化培训方案
  - 优先考虑教情分析的针对性建议

#### 3.5.2 智能决策规则

##### 知识维度决策规则：
- **知识短板识别规则**：
  - IF 教情分析显示知识维度存在短板（如"5以内数的认识和加减法"偏态量为+）THEN 推荐"学科知识教学策略"相关培训
  - 优先级：高
  - 适用场景：基础知识点教学存在困难

- **知识长板应用规则**：
  - IF 教情分析显示知识维度存在长板（如"认识图形"偏态量为-）THEN 策略建议："教研上需关注教师教学的方法策略"
  - 推荐"教学方法优化"培训
  - 优先级：中
  - 适用场景：知识掌握良好，但教学方法需优化

##### 技能维度决策规则：
- **技能短板强化规则**：
  - IF 教情分析显示技能维度存在短板（如"推理、数学表达"偏态量为-）THEN 推荐"学生推理能力培养""数学表达能力提升"相关培训
  - 优先级：高
  - 策略："教师教学的成效好，只需有针对性的加强学生的学、练即可"
  - 适用场景：学生技能培养存在困难

- **技能长板维持规则**：
  - IF 教情分析显示技能维度存在长板（如"认知、运算、操作"偏态量为-）THEN 策略建议："教师的教学成效很好"
  - 保持现有培训节奏
  - 优先级：低
  - 适用场景：技能教学效果良好

##### 能力素养维度决策规则：
- **能力短板提升规则**：
  - IF 教情分析显示能力素养维度存在短板（如"逻辑思维"偏态量为-）THEN 推荐"逻辑思维能力培养"相关培训
  - 优先级：高
  - 策略："教师要在如何提高整体逻辑思维的能力素养水平上下功夫"
  - 适用场景：学生能力素养培养存在困难

- **能力长板深化规则**：
  - IF 教情分析显示能力素养维度存在长板（如"直观想象、数理关系"偏态量为+/-）THEN 策略建议："教研上需关注数理关系的方法策略"
  - 推荐"教学方法深化"培训
  - 优先级：中
  - 适用场景：能力培养效果良好，可进一步深化

#### 3.5.3 优先级判定规则
- **高优先级**：能力短板、紧急需求、政策要求、教学效果差
- **中优先级**：能力提升、专业发展、兴趣导向、教学效果一般
- **低优先级**：维持现状、巩固优势、非紧急需求、教学效果良好

#### 3.5.4 匹配度计算规则
- **基于能力差距的匹配度**：
  - 匹配度 = (能力差距分数 / 最大可能差距) * 0.6 + (政策符合度分数 / 最大政策分数) * 0.4
  - 差距越大，匹配度越高

- **基于培训历史的匹配度**：
  - IF 培训项目已参与且通过 THEN 匹配度降低50%
  - IF 培训项目已参与但未通过 THEN 匹配度降低20%
  - IF 培训项目未参与 THEN 保持原始匹配度

- **基于教情分析的匹配度**：
  - IF 培训内容与教情分析建议一致 THEN 匹配度增加30%
  - IF 培训内容与教情分析建议部分一致 THEN 匹配度增加15%

#### 3.5.5 决策约束规则
- **重复推荐约束**：
  - IF 某培训项目已获得证书 THEN 不再推荐相同或相似培训
  - IF 培训学时要求与教师当前需求不匹配 THEN 调整推荐或提供说明

- **级别匹配约束**：
  - IF 培训级别与教师当前发展阶段不匹配 THEN 提供适合级别的替代推荐
  - IF 培训内容难度与教师能力不匹配 THEN 调整推荐级别

- **时间约束**：
  - IF 培训时间与教师已有培训时间冲突 THEN 提供时间调整建议
  - IF 培训频率过高 THEN 建议分散安排

#### 3.5.6 可视化规则
- **模块顺序规则**：
  - 按固定模块顺序排版：个人信息→专业发展路径→当前阶段分析→各维度数据→培训画像→智能决策→教研能力评估

- **图表类型规则**：
  - 专业发展路径：线性路径图，5个阶段
  - 多维度对比：柱状图，个人/区域/常模值对比
  - 能力分析：雷达图，5维度分布
  - 培训分布：饼图，培训级别分布
  - 历史趋势：折线图，学时分布

- **交互规则**：
  - 支持在线查看和PDF导出
  - 支持数据筛选和排序
  - 支持图表放大查看

## 4. 技术要求

- 支持PDF格式的教情分析表解析，提取结构化数据
- 实现高效的数据处理和分析算法，支持大数据量处理
- 提供响应式的可视化界面，适配不同设备
- 确保数据安全和隐私保护，符合相关法规要求
- 支持实时数据更新和动态报告生成
- 实现高可用性和高并发处理能力
- 支持多租户架构，满足不同机构需求
